# 常用状态码

HTTP 状态码是服务器在响应客户端请求时返回的三位数字代码，它们就像是服务器在对话中使用的“快捷短语”，告诉你请求是成功、需要进一步操作、出错或是其他情况。

| 状态码                    | 类别    | 名称                             | 主要使用场景                                                                                                                                                |
| ---------------------- | ----- | ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| ​**​1xx - 信息性状态码​**​   |       |                                | <p>表示请求已被接收，需要继续处理 <br><br>。</p>                                                                                                                      |
| `100`                  | 信息性   | Continue                       | <p>客户端在发送较大请求体前，征询服务器意见，服务器同意接收 <br><br>。</p>                                                                                                         |
| `101`                  | 信息性   | Switching Protocols            | <p>服务器应客户端请求切换协议，如升级到 WebSocket <br><br>。</p>                                                                                                         |
| ​**​2xx - 成功状态码​**​    |       |                                | <p>表示请求已被服务器成功接收、理解并处理 <br><br>。</p>                                                                                                                  |
| `200`                  | 成功    | OK                             | <p>​<strong>​标准成功响应​</strong>​，请求成功，响应体中包含所需资源 <br><br>。</p>                                                                                          |
| `201`                  | 成功    | Created                        | <p>​<strong>​资源创建成功​</strong>​，通常用于 <code>POST</code>或 <code>PUT</code>请求，响应头应包含新资源的 URI（Location）<br><br>。</p>                                       |
| `204`                  | 成功    | No Content                     | <p>​<strong>​请求成功，但无返回内容​</strong>​。常用于 <code>DELETE</code>请求成功或 <code>PUT/POST</code>更新后无需返回数据 <br><br>。</p>                                         |
| `206`                  | 成功    | Partial Content                | <p>​<strong>​部分内容​</strong>​。客户端进行了范围请求（Range头），服务器成功返回部分资源，用于断点续传或大文件分块下载 <br><br>。</p>                                                              |
| ​**​3xx - 重定向状态码​**​   |       |                                | <p>表示需要客户端执行进一步的操作才能完成请求 <br><br>。</p>                                                                                                                |
| `301`                  | 重定向   | Moved Permanently              | <p>​<strong>​永久重定向​</strong>​。请求的资源已永久移动到新位置，搜索引擎会更新索引。用户应更新书签 <br><br>。</p>                                                                          |
| `302`                  | 重定向   | Found                          | <p>​<strong>​临时重定向​</strong>​。资源临时从不同 URI 响应，客户端​<strong>​应​</strong>​继续使用原始 URI。但实际应用中，许多浏览器会将其作为 303 处理 <br><br>。</p>                               |
| `304`                  | 重定向   | Not Modified                   | <p>​<strong>​资源未修改​</strong>​。当客户端发起条件请求（如携带 <code>If-Modified-Since</code>头）时，如果资源未变，服务器返回此状态，告知客户端可继续使用缓存 <br><br>。</p>                             |
| `307`/ `308`           | 重定向   | Temporary / Permanent Redirect | <p>​<strong>​临时/永久重定向​</strong>​。与 302/301 类似，但​<strong>​严格要求​</strong>​客户端在重定向时​<strong>​不得​</strong>​改变原有的 HTTP 方法（如 POST 重定向后仍为 POST）<br><br>。</p> |
| ​**​4xx - 客户端错误状态码​**​ |       |                                | <p>表示客户端的请求有错误，服务器无法或不会处理该请求 <br><br>。</p>                                                                                                            |
| `400`                  | 客户端错误 | Bad Request                    | <p>​<strong>​请求报文存在语法错误​</strong>​。例如，JSON 格式错误、缺少必要参数、参数类型不匹配等 <br><br>。</p>                                                                         |
| `401`                  | 客户端错误 | Unauthorized                   | <p>​<strong>​未认证​</strong>​。请求需要用户认证，但未能提供有效的身份凭证。响应通常包含 <code>WWW-Authenticate</code>头告知如何认证 <br><br>。</p>                                           |
| `403`                  | 客户端错误 | Forbidden                      | <p>​<strong>​已认证，但无权限​</strong>​。服务器理解请求，但拒绝执行，通常因为权限不足 <br><br>。</p>                                                                                 |
| `404`                  | 客户端错误 | Not Found                      | <p>​<strong>​资源未找到​</strong>​。服务器上不存在所请求的资源 <br><br>。</p>                                                                                             |
| `429`                  | 客户端错误 | Too Many Requests              | <p>​<strong>​请求过于频繁​</strong>​。客户端在给定时间内发送了太多请求，触发了服务器的速率限制。响应头可包含 <code>Retry-After</code>提示重试时间 <br><br>。</p>                                       |
| ​**​5xx - 服务器错误状态码​**​ |       |                                | <p>表示服务器在处理请求的过程中发生了错误 <br><br>。</p>                                                                                                                  |
| `500`                  | 服务器错误 | Internal Server Error          | <p>​<strong>​通用服务器内部错误​</strong>​。一个笼统的错误代码，表示服务器遇到了意外情况，无法完成请求 <br><br>。</p>                                                                         |
| `502`                  | 服务器错误 | Bad Gateway                    | <p>​<strong>​错误网关​</strong>​。当服务器作为网关或代理时，从上游服务器收到了一个无效的响应 <br><br>。</p>                                                                              |
| `503`                  | 服务器错误 | Service Unavailable            | <p>​<strong>​服务不可用​</strong>​。服务器当前由于超载、停机维护等原因无法处理请求，这通常是临时状态。响应头可包含 <code>Retry-After</code>估计服务恢复时间 <br><br>。</p>                                  |
| `504`                  | 服务器错误 | Gateway Timeout                | <p>​<strong>​网关超时​</strong>​。服务器作为网关或代理，未能及时从上游服务器收到响应 <br><br>。</p>                                                                                  |
